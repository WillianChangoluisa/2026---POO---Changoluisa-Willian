🎯 CONCEPTOS DE POO APLICADOS

1️⃣  CONSTRUCTOR TRADICIONAL
    └─ def __init__(self, nombre, categoria, precio, disponible=True)
       → Inicializa atributos privados
       → Utiliza setters para validación

2️⃣  DECORADOR @property
    └─ @property
       @nombre.setter
       → Lectura controlada de atributos
       → Escritura con validación automática

3️⃣  DECORADOR @setter
    └─ @precio.setter
       → Valida antes de asignar
       → Lanza excepciones para datos inválidos

4️⃣  DECORADOR @dataclass
    └─ @dataclass
       → Genera automáticamente __init__(), __repr__(), __eq__()
       → Reduce código boilerplate
       → Validaciones en __post_init__()

5️⃣  ENCAPSULACIÓN
    └─ Atributos privados (_nombre, _precio, _categoria)
       → Protege datos internos
       → Acceso controlado mediante decoradores

6️⃣  VALIDACIÓN DE ENTRADA
    └─ Todos los setters incluyen validaciones
       → Nombre no vacío
       → Categoría no vacía
       → Precio > 0
       → Correo válido (contiene @)
       → Sin duplicados

7️⃣  ARQUITECTURA POR CAPAS
    └─ CAPA DE DATOS       (modelos/)
       CAPA DE NEGOCIO     (servicios/)
       CAPA DE PRESENTACIÓN (main.py)
       → Separación de responsabilidades
       → Bajo acoplamiento
       → Fácil de mantener y extender

8️⃣  MANEJO DE EXCEPCIONES
    └─ try/except para errores
       ValueError para validaciones
       Mensajes claros y descriptivos
